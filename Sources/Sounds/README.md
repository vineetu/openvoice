# Sounds

Five bundled chimes Jot plays on recorder / delivery events.

These WAVs were generated procedurally — we don't have licensed sound
assets, so each chime is a short sine-wave envelope shaped to read as a
distinct "start / stop / cancel / complete / error" cue.

## Files

| File                          | Shape                                                           |
|-------------------------------|-----------------------------------------------------------------|
| `recording-start.wav`         | 120 ms 880 Hz sine, 20 ms fade-in/out                           |
| `recording-stop.wav`          | 120 ms 660 Hz sine, 20 ms fade-in/out                           |
| `recording-cancel.wav`        | 80 ms descending 440 → 220 Hz ramp                              |
| `transcription-complete.wav`  | 180 ms — 523 Hz (C5) then 659 Hz (E5) eighth-note pair          |
| `error.wav`                   | 200 ms 180 Hz with a small amount of white noise                |

All are 44.1 kHz, 16-bit PCM, mono, under ~18 KB each.

## Regenerating

The five WAVs are generated by a one-shot Swift script kept outside the
app target (it's not shipped). To rebuild them:

```bash
swift /tmp/gen_chimes.swift
```

The script (mirrored in the Phase 5 Polish PR description) writes the
five WAV files directly into this directory. It's idempotent — it just
overwrites whatever is here. If the generator is lost, regenerating it
from the table above is a ~50-line job (header + sine + fade).

## Why procedural?

- No licensing concerns.
- Trivial to regenerate or tweak.
- Small enough that the bundle footprint stays negligible.

If we later license proper chimes (e.g. a designer delivers a set), the
only thing that changes is the five files here — `SoundEffect.fileName`
already points at stable names.
